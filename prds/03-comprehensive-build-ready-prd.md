# 3. The comprehensive build-ready PRD — waterfall-leaning, spec-complete process

A full document: personas, detailed user stories with acceptance criteria,
functional + non-functional requirements, wireframes, success metrics, timeline.
Detailed enough that engineers can build with no further clarification.

- **Process it embodies:** specify thoroughly up front, reduce ambiguity and
  mid-build churn. Heavier, but high-fidelity.
- **Best for:** larger orgs, regulated domains, cross-team dependencies,
  contractor/outsourced builds.
