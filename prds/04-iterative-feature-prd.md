# 4. Real feature PRDs from shipping companies — iterative / agile feature process

Concrete, scoped specs for a single feature on an existing product — e.g.
Medium's free-trial proposal, Webflow's page-branching spec, Stripe's
new-payment-method support. Tightly scoped, metric-driven, built on top of an
existing product.

- **Process it embodies:** continuous delivery — each PRD is a bounded increment
  with its own hypothesis and metric, not a whole product.
- **Best for:** mature products shipping a steady stream of improvements.
