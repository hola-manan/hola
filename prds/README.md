# PRD process styles

Five strong PRD examples, each representing a **different process** — not just a
different layout, but a different way of thinking about and building the product.

| # | Style | Process | Best for |
|---|-------|---------|----------|
| 01 | [Amazon Working Backwards (PR/FAQ)](01-amazon-working-backwards-prfaq.md) | Narrative / customer-first | New bets, 0→1, exec buy-in |
| 02 | [Lean one-pager](02-lean-one-pager.md) | Lightweight / fast-iteration | Startups, small features |
| 03 | [Comprehensive build-ready PRD](03-comprehensive-build-ready-prd.md) | Waterfall-leaning, spec-complete | Enterprise, regulated, hand-offs |
| 04 | [Iterative feature PRD](04-iterative-feature-prd.md) | Agile / continuous delivery | Mature products shipping increments |
| 05 | [AI / ML product spec](05-ai-ml-product-spec.md) | Experiment-driven, probabilistic | LLM/ML features |

## Quick way to choose
- New bet needing buy-in → **01**
- Move fast / small scope → **02**
- High-stakes or hand-off build → **03**
- Ongoing feature work → **04**
- AI/ML feature → **05**

The [prompt sequence](../prompts) adapts to any of these — just name the target
format in Prompt 3 (e.g. "format this as an Amazon PR/FAQ").
